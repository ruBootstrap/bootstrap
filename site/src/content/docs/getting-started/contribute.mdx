## Настройка инструментов

Bootstrap использует [скрипты npm](https://docs.npmjs.com/misc/scripts/) для сборки документации и компиляции исходных файлов. Наш [package.json]([[config:repo]]/blob/v[[config:current_version]]/package.json) содержит эти скрипты для компиляции кода, запуска тестов и многого другого. Они не предназначены для использования вне нашего репозитория и документации.

Чтобы использовать нашу систему сборки и запустить нашу документацию локально, вам понадобится копия исходных файлов Bootstrap и Node. Выполните следующие шаги, и вы будете готовы к работе:

1. [Загрузите и установите Node.js](https://nodejs.org/en/download/), который мы используем для управления нашими зависимостями.
2. Либо [загрузите исходники Bootstrap]([[config:download.source]]), либо сделайте форк и клонируйте [репозиторий Bootstrap]([[config:repo]]).

3. Перейдите в корневой каталог `/bootstrap` и запустите `npm install`, чтобы установить наши локальные зависимости, перечисленные в [package.json]([[config:repo]]/blob/v[[config:current_version]]/package.json).

После завершения вы сможете запускать различные команды, предоставленные из командной строки.

## Использование скриптов npm

Наш [package.json]([[config:repo]]/blob/v[[config:current_version]]/package.json) включает в себя многочисленные задачи для разработки проекта. Запустите `npm run`, чтобы увидеть все скрипты npm в вашем терминале. **Основные задачи включают:**

<BsTable>
| Задача | Описание |
| --- | --- |
| `npm start` | Компилирует CSS и JavaScript, собирает документацию и запускает локальный сервер. |
| `npm run dist` | Создает каталог `dist/` со скомпилированными файлами. Использует [Sass](https://sass-lang.com/), [Autoprefixer](https://github.com/postcss/autoprefixer) и [terser](https://github.com/terser/terser). |
| `npm test` | Запускает тесты локально после запуска `npm run dist` |
| `npm run docs-serve` | Собирает и запускает документацию локально. |
</BsTable>

## Sass

Bootstrap использует [Dart Sass](https://sass-lang.com/dart-sass/) для компиляции наших исходных файлов Sass в файлы CSS (включены в наш процесс сборки), и мы рекомендуем вам сделать то же самое, если вы компилируете Sass с помощью собственного конвейера ресурсов. Ранее мы использовали Node Sass для Bootstrap v4, но LibSass и пакеты, созданные на его основе, включая Node Sass, теперь [устарели](https://sass-lang.com/blog/libsass-is-deprecated/).

Dart Sass использует точность округления 10 и из соображений эффективности не позволяет изменять это значение. Мы не снижаем эту точность во время дальнейшей обработки нашего сгенерированного CSS, например, во время минификации, но если вы решили это сделать, мы рекомендуем поддерживать точность не менее 6, чтобы предотвратить проблемы с округлением браузера.

## Автопрефиксер

Bootstrap использует [Autoprefixer](https://github.com/postcss/autoprefixer) (включенный в наш процесс сборки) для автоматического добавления префиксов поставщиков к некоторым свойствам CSS во время сборки. Это экономит время и код, позволяя писать ключевые части нашего CSS за один раз, устраняя при этом необходимость в миксинах от поставщиков, подобных тем, что есть в v3.

Мы поддерживаем список браузеров, поддерживаемых Autoprefixer, в отдельном файле в нашем репозитории GitHub. Подробности см. в [.browserslistrc]([[config:repo]]/blob/v[[config:current_version]]/.browserslistrc).

## RTLCSS

Bootstrap использует [RTLCSS](https://rtlcss.com/) для обработки скомпилированного CSS и преобразования их в RTL - в основном заменяя свойства, учитывающие горизонтальное направление (например, `padding-left`), их противоположностью. Это позволяет нам писать наш CSS только один раз и вносить незначительные изменения с помощью RTLCSS директив [control](https://rtlcss.com/learn/usage-guide/control-directives/) и [value](https://rtlcss.com/learn/usage-guide/value-directives/).

## Локальная документация

Для локального запуска нашей документации требуется использование Astro. Astro — это современный статический генератор сайтов, который позволяет нам создавать нашу документацию с помощью комбинации компонентов Markdown и React. Вот как это сделать:

1. Запустите [настройку инструментария](#tooling-setup) выше, чтобы установить все зависимости.
2. Из корневого каталога `/bootstrap` запустите `npm run docs-serve` в командной строке.
3. Откройте `http://localhost:4321/` в своем браузере, и вуаля.

Узнайте больше об использовании Astro, прочитав его [документацию](https://docs.astro.build/en/getting-started/).

## Исправление проблем

Если у Вас возникнут проблемы с установкой зависимостей, удалите все предыдущие версии зависимостей (глобальные и локальные). Затем перезапустите `npm install`.
