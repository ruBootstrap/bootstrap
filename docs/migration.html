Переход с 2.x на 3.0

Bootstrap 3 не обратно совместим с v2.x. Используйте этот раздел как общее руководство по обновлению с v2.x до v3.0. Для более подробного обзора смотрите что нового в анонсе релиза v3.0.

Дополнительные замечания

Другие изменения в v3.0 не так очевидны. Базовые классы, ключевые стили и поведения были изменены для гибкости и подхода mobile first. Вот частичный список:

  • По умолчанию текстовые элементы форм теперь имеют минимальное оформление. Для цветов фокуса и скруглённых углов применяйте класс .form-control к элементу.
  • Текстовые элементы форм с классом .form-control теперь по умолчанию занимают 100% ширины. Оборачивайте input в <div class="col-*"></div> для управления шириной.
  • .badge больше не имеет контекстных классов (-success, -primary и т.д.).
  • .btn теперь также должен использовать .btn-default для получения "обычной" кнопки.
  • .row теперь всегда fluid.
  • Изображения больше не адаптивны по умолчанию. Используйте .img-responsive для fluid-<img>.
  • Иконки теперь .glyphicon и являются шрифтовыми. Для иконок требуется базовый икон-класс (например, .glyphicon .glyphicon-asterisk).
  • Typeahead удалён, используйте Twitter Typeahead.
  • Разметка модальных окон значительно изменилась. .modal-header, .modal-body и .modal-footer теперь обёрнуты в .modal-content и .modal-dialog для лучшей мобильной стилизации и поведения. Также больше не нужно применять .hide к .modal в разметке.
  • С версии v3.1.0 HTML, загружаемый через опцию remote модального окна, теперь вставляется в .modal-content (с v3.0.0 по v3.0.3 — в .modal), а не только в .modal-body. Это позволяет легко менять заголовок и футер модального окна, а не только тело.
  • Чекбоксы и радио-кнопки в button.js теперь используют data-toggle="buttons" вместо data-toggle="buttons-checkbox" или data-toggle="buttons-radio" в разметке.
  • JavaScript-события теперь имеют namespace. Например, для обработки события "show" модального окна используйте 'show.bs.modal'. Для вкладок "shown" — 'shown.bs.tab' и т.д.

Для дополнительной информации по обновлению до v3.0 и примеров кода от сообщества смотрите Bootply.
